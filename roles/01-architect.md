# 角色任务卡: 架构师 (Architect)

## 角色信息
- **角色ID**: 01-architect
- **角色名称**: 系统架构师
- **预计时长**: 16 小时
- **主要职责**: 整体架构设计、技术规范、接口定义

## 核心任务

### 1. 架构设计文档

创建完整的技术架构文档：
- 系统架构图
- 技术栈选择与理由
- 模块划分与职责
- 数据流设计
- 接口规范定义

**输出**: `docs/ARCHITECTURE.md`

### 2. 插件系统设计

设计可扩展的插件系统：
- 插件接口定义
- 插件生命周期管理
- 插件注册与发现机制
- 配置schema定义

**输出**: `docs/PLUGIN_SYSTEM.md`

### 3. 数据模型设计

设计核心数据模型：
- 实验模型
- 追踪事件模型
- 转化模型
- 结算模型
- 各插件数据模型

**输出**: `docs/DATA_MODELS.md`

### 4. API 接口规范

定义 RESTful API 规范：
- 端点定义
- 请求/响应格式
- 错误处理规范
- 认证授权机制

**输出**: `docs/API_SPEC.md`

### 5. 数据库设计

设计数据库表结构：
- 核心表设计
- 索引策略
- 关系定义
- 迁移策略

**输出**: 交给数据库工程师实施

### 6. 部署架构设计

设计云部署架构：
- 服务拆分策略
- 云平台选择
- 网络拓扑
- 监控与日志

**输出**: `docs/DEPLOYMENT_ARCHITECTURE.md`
