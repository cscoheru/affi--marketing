## 角色信息
- **角色ID**: 07-product-manager
- **角色名称**: 产品经理
- **预计时长**: 16 小时
- **主要职责**: 需求分析、用户反馈、功能规划

## 📋 核心任务

### 1. 用户反馈收集 (4h)

**任务**: 收集并分析第一阶段用户反馈

**活动**:
- 分析用户行为数据 (如果有)
- 整理用户反馈意见
- 识别常见问题和痛点
- 收集功能请求

**输出**: `docs/PHASE2_REQUIREMENTS.md` - 第二阶段需求文档

---

### 2. 数据分析 (3h)

**任务**: 分析第一阶段使用数据

**活动**:
- 分析页面访问数据
- 分析 API 调用统计
- 识别使用瓶颈
- 发现优化机会

**输出**: 数据分析报告 (添加到需求文档)

---

### 3. 功能优先级规划 (4h)

**任务**: 制定第二阶段功能优先级

**活动**:
- 列出所有潜在功能
- 评估每个功能的价值和成本
- 使用 RICE 或 ICE 评分
- 确定开发优先级

**输出**: `docs/PHASE2_BACKLOG.md` - 功能待办清单

---

### 4. 用户故事编写 (3h)

**任务**: 为优先功能编写用户故事

**活动**:
- 编写用户故事 (As a... I want... So that...)
- 定义验收标准
- 绘制原型流程图
- 准备开发需求

**输出**: `docs/USER_STORIES.md` - 用户故事文档

---

### 5. 跨会话协调 (2h)

**任务**: 与其他会话沟通需求

**活动**:
- 向前端优化会话传达 UI 需求
- 向后端优化会话传达性能目标
- 向数据工程会话传达分析需求
- 解答各会疑问

**输出**: `docs/MESSAGES/` 中的协调记录
