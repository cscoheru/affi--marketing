### 1. 素材上传错误处理改进 ✅

**问题**: 上传文件时报错"请求失败"，错误信息不详细

**修复内容**:
- 改进了 `lib/api.ts` 中的 `postForm` 方法
- 增强了错误处理逻辑，提供更详细的错误信息
- 添加了JSON解析失败的fallback处理

**修复位置**: `lib/api.ts:111-132`

**修复前**:
```typescript
if (!response.ok) {
  const error = await response.json().catch(() => ({ message: '请求失败' }))
  throw new Error(error.message || '请求失败')
}
```

**修复后**:
```typescript
if (!response.ok) {
  let errorMessage = '请求失败'
  try {
    const errorData = await response.json().catch(() => null)
    if (errorData) {
      errorMessage = errorData.message || errorData.error || errorMessage
    }
  } catch {
    errorMessage = `请求失败 (${response.status})`
  }
  throw new Error(errorMessage)
}
```

## 常见问题

### Q1: AI服务一直显示离线怎么办？

**A**: 检查以下几点：
1. 确认 `NEXT_PUBLIC_AI_URL` 配置正确
2. 访问 `https://ai-api.zenconsult.top/health` 检查服务状态
3. 如果服务在线但前端显示离线，检查网络连接
4. 查看浏览器控制台是否有CORS错误

### Q2: 素材上传还是报错怎么办？

**A**:
1. 打开浏览器开发者工具查看网络请求
2. 检查上传API的响应内容
3. 确认后端API服务是否正常运行
4. 检查文件大小是否超过限制

### Q3: 生成的内容质量不理想？

**A**:
1. 尝试切换不同的AI模型
2. 调整语调风格和内容长度
3. 在主题中添加更具体的描述
4. 使用"重新生成"按钮尝试其他结果

### Q4: 如何查看AI服务的详细日志？

**A**:
1. 打开浏览器控制台
2. 查看网络请求标签
3. 找到 `https://ai-api.zenconsult.top/api/v1/generate/blog-content` 请求

**问题**: 素材库显示 "NaN undefined"

**修复内容** (`app/(content)/materials/page.tsx`):
- 修复 `formatFileSize` 函数处理边界情况
  - 添加 `undefined`、`null`、`NaN` 检查
  - 无效值时显示 "未知大小"
- 修复日期显示处理
  - 添加 `material.createdAt` 空值检查
  - 无效日期时显示 "未知日期"

**修复前**:
```typescript
function formatFileSize(bytes: number): string {
  if (bytes === 0) return '0 B'
  const k = 1024
  const sizes = ['B', 'KB', 'MB', 'GB']
  const i = Math.floor(Math.log(bytes) / Math.log(k))
  return `${(bytes / Math.pow(k, i)).toFixed(1)} ${sizes[i]}`
}
```

**修复后**:
```typescript
function formatFileSize(bytes: number | undefined | null): string {
  if (bytes === undefined || bytes === null || isNaN(bytes) || bytes === 0) return '未知大小'
  const k = 1024
  const sizes = ['B', 'KB', 'MB', 'GB']
  const i = Math.floor(Math.log(bytes) / Math.log(k))
  return `${(bytes / Math.pow(k, i)).toFixed(1)} ${sizes[i]}`
}
```
