# 会话协作快速指南

**版本**: 1.1.0
**更新**: 2026-03-03

---

## 🎯 新会话必读

**你是新会话？请从这里开始：**

1. **首先** 阅读 `docs/SESSION_ONBOARDING.md` ← 📍 会话入职指南
   - 项目完整架构
   - 代码位置和结构
   - 部署信息和硬件配置
   - 第一阶段成果

2. **然后** 阅读 `docs/COLLABORATION_QUICK_START.md` ← 📍 你在这里
   - 协作流程
   - 常用操作

3. **接着** 查看 `docs/SESSION_REGISTRY.md`
   - 当前会话状态

4. **最后** 查看你的任务卡 `roles/XX-角色.md`
   - 具体任务内容

---

## 🎯 核心原则

1. **会话独立** - 每个会话独立工作，自主决策
2. **文档通信** - 通过文件共享信息，不直接访问其他会话
3. **状态透明** - 及时更新状态，让其他会话了解进度
4. **依赖明确** - 清晰声明依赖关系，避免阻塞

## 📋 会话启动清单

### 新会话启动时

- [ ] 1. 阅读 `docs/SESSION_REGISTRY.md` 了解项目状态
- [ ] 2. 阅读 `docs/SESSION_COLLABORATION.md` 了解协作规则
- [ ] 3. 检查 `docs/MESSAGES_{会话ID}.md` 是否有留言
- [ ] 4. 检查 `docs/API_CHANGES.md` 是否有 API 变更
- [ ] 5. 检查 `docs/SCHEMA_CHANGES.md` 是否有数据变更

### 会话工作期间

- [ ] 每完成一个任务，更新 `docs/SESSION_LOG.md`
- [ ] 如有 API 变更，提交 `docs/API_CHANGES.md`
- [ ] 如有数据变更，提交 `docs/SCHEMA_CHANGES.md`
- [ ] 如需其他会话配合，发送消息到 `docs/MESSAGES_{目标会话}.md`

### 会话结束时

- [ ] 更新 `docs/SESSION_LOG.md` 记录完成状态
- [ ] 通知依赖会话任务完成
- [ ] 清理 `docs/MESSAGES_{会话ID}.md` 中的待处理消息

## 🚨 常见场景

### 场景1: 我需要修改 API 接口

1. 在 `docs/API_CHANGES.md` 创建变更提案
2. 等待依赖会话确认 (查看 `docs/MESSAGES_{你的会话}.md`)
3. 实施变更
4. 更新 `docs/API_SPEC.md`

### 场景2: 我被其他会话阻塞了

1. 在 `docs/SESSION_LOG.md` 记录阻塞
2. 在 `docs/MESSAGES_{阻塞会话}.md` 发送消息
3. 等待响应或上报项目经理

### 场景3: 我完成了任务

1. 在 `docs/SESSION_LOG.md` 记录完成
2. 检查是否有下游依赖会话
3. 如有，通知他们任务完成

### 场景4: 我需要其他会话的信息

1. 查看 `docs/SESSION_REGISTRY.md` 了解会话状态
2. 在 `docs/MESSAGES_{目标会话}.md` 发送消息
3. 等待响应

## 💡 最佳实践

### DO ✅
- 及时更新状态
- 文档先行
- 尊重依赖
- 主动沟通

### DON'T ❌
- 不要直接修改他人代码
- 不要在未通知的情况下变更 API
- 不要忽略其他会话的消息
- 不要阻塞超过 24h 不上报
