## 1. 系统概述

Affi-Marketing 是一个可扩展的商业模式测试与统一管理平台，支持多种电商流量变现模式的验证和管理。

### 1.1 支持的业务模式

| 模式 | 描述 |
|------|------|
| **程序化 SEO + 联盟营销** | 自动化内容生成 + 联盟链接变现 |
| **GEO (生成式引擎优化)** | AI 生成内容优化搜索引擎排名 |
| **AI 代理电商** | AI Agent 辅助的电商导购 |
| **自建联盟营销 SaaS** | 完整的联盟营销管理系统 |

### 1.2 核心功能

- **实验管理**: 创建、配置、监控多个商业模式实验
- **流量追踪**: 多触点归因追踪和转化分析
- **插件系统**: 可扩展的功能模块架构
- **AI 集成**: 智能内容生成和优化
- **数据结算**: 自动化的收益结算和分成

## 3. 核心模块

### 3.1 前端控制台 (Vue 3)

**职责**: 用户交互界面，实验管理和数据可视化

| 模块 | 功能 |
|------|------|
| **仪表板** | 实验概览、关键指标、趋势图表 |
| **实验管理** | 创建/编辑/删除实验，配置参数 |
| **数据看板** | 流量分析、转化漏斗、归因报告 |
| **插件管理** | 插件启用/禁用、参数配置 |
| **系统设置** | 用户管理、权限配置、系统参数 |

**路由结构**:
```
/                          → 仪表板
/experiments               → 实验列表
/experiments/create        → 创建实验
/experiments/:id           → 实验详情
/analytics                 → 数据分析
/analytics/traffic         → 流量分析
/analytics/conversions     → 转化分析
/plugins                   → 插件管理
/settings                  → 系统设置
```

### 3.2 后端 API (Go + Gin)

**职责**: 核心业务逻辑、插件调度、数据管理

| 模块 | 功能 |
|------|------|
| **实验管理** | 实验 CRUD、状态管理、配置存储 |
| **追踪服务** | 事件接收、数据处理、实时统计 |
| **归因引擎** | 多触点归因、路径分析、贡献计算 |
| **结算服务** | 收益计算、分成处理、账单生成 |
| **插件系统** | 插件加载、执行调度、结果聚合 |
| **AI 代理** | AI 服务通信、请求队列、结果缓存 |

**API 结构**:
```
/api/v1/
├── /experiments          → 实验管理
├── /tracking             → 追踪事件
├── /analytics            → 数据分析
├── /attribution          → 归因分析
├── /settlement           → 结算管理
├── /plugins              → 插件管理
└── /system               → 系统管理
```

### 3.3 AI 服务 (FastAPI)

**职责**: 智能内容生成、SEO 优化、智能推荐

| 模块 | 功能 |
|------|------|
| **内容生成** | 文章生成、标题优化、描述改写 |
| **SEO 优化** | 关键词研究、元标签生成、内容结构化 |
| **智能推荐** | 产品推荐、内容匹配、用户画像 |
| **模型适配** | 多模型支持、成本优化、性能监控 |

### 3.4 追踪服务 (Cloudflare Workers)

**职责**: 边缘流量追踪、实时数据处理

| 功能 | 描述 |
|------|------|
| **脚本注入** | 自动注入追踪代码到目标网站 |
| **事件捕获** | 页面浏览、点击、转化等事件 |
| **实时处理** | 边缘计算，减少延迟 |
| **离线支持** | Service Worker 缓存机制 |

## 4. 数据流设计

### 4.1 用户访问流程

```
用户访问网站
    │
    ├─→ 1. Cloudflare Workers 拦截
    │       │
    │       ├─→ 读取追踪参数 (UTM、Referrer 等)
    │       ├─→ 识别用户身份 (Cookie/Fingerprint)
    │       ├─→ 记录访问事件 (异步发送到后端)
    │       └─→ 注入追踪脚本
    │
    ├─→ 2. 用户浏览页面
    │       │
    │       ├─→ 追踪脚本收集行为数据
    │       ├─→ 发送事件到 Workers
    │       └─→ Workers 批量发送到后端
    │
    ├─→ 3. 用户产生转化
    │       │
    │       ├─→ 触发转化事件
    │       ├─→ 归因引擎分析转化路径
    │       ├─→ 计算各触点贡献
    │       └─→ 触发结算流程
    │
    └─→ 4. 管理员查看数据
            │
            ├─→ 前端请求后端 API
            ├─→ 后端查询 PostgreSQL/Redis
            ├─→ 聚合分析数据
            └─→ 返回可视化结果
```

### 4.2 AI 内容生成流程

```
管理员创建 SEO 实验
    │
    ├─→ 1. 配置实验参数
    │       │
    │       ├─→ 选择目标关键词
    │       ├─→ 设置内容类型
    │       └─→ 配置生成策略
    │
    ├─→ 2. 后端调用 AI 服务
    │       │
    │       ├─→ 发送生成任务
    │       ├─→ AI 服务处理请求
    │       │   │
    │       │   ├─→ 选择合适的模型
    │       │   ├─→ 构建提示词
    │       │   ├─→ 调用 LLM API
    │       │   └─→ 后处理优化
    │       │
    │       └─→ 返回生成结果
    │
    ├─→ 3. 内容发布
    │       │
    │       ├─→ 存储到 MinIO
    │       ├─→ 更新实验状态
    │       └─→ 通知发布渠道
    │
    └─→ 4. 效果监控
            │
            ├─→ 追踪搜索引擎收录
            ├─→ 监控流量变化
            └─→ 优化生成策略
```

## 5. 插件系统架构

### 5.1 插件接口定义

```go
// Plugin 插件基础接口
type Plugin interface {
    // Init 初始化插件
    Init(config PluginConfig) error

    // Execute 执行插件逻辑
    Execute(ctx context.Context, input PluginInput) (PluginOutput, error)

    // Shutdown 清理资源
    Shutdown() error

    // Info 获取插件信息
    Info() PluginInfo
}

// PluginInfo 插件元信息
type PluginInfo struct {
    ID          string
    Name        string
    Version     string
    Description string
    Author      string
    Type        PluginType
}

// PluginType 插件类型
type PluginType string

const (
    PluginTypeSEO         PluginType = "seo"          // SEO 插件
    PluginTypeAttribution PluginType = "attribution"  // 归因插件
    PluginTypeSettlement   PluginType = "settlement"   // 结算插件
    PluginTypeTracking     PluginType = "tracking"     // 追踪插件
    PluginTypeAI           PluginType = "ai"           // AI 插件
)
```

### 5.2 插件生命周期

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  加载    │ → │  初始化  │ → │  执行    │ → │  清理    │
│  Load   │    │  Init   │    │ Execute │    │ Shutdown│
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │
     ▼              ▼              ▼              ▼
 动态加载      读取配置       业务逻辑       释放资源
 验证接口      建立连接       处理数据       关闭连接
 注册插件      初始化状态     返回结果       清理缓存
```
