# 联盟营销内容系统整体设计

## 1. 核心认知

**内容是联盟营销的引擎**

```
产品 → 内容 → 发布 → 转化
  ↑                      │
  └──────────────────────┘
         数据反馈
```

- 没有优质内容，就没有流量
- 没有流量，就没有转化
- 产品再好，没有内容支撑也是零

## 2. 三个模块的关系矩阵

| 关系 | 产品库 | 内容库 | 发布中心 |
|-----|-------|-------|---------|
| **产品库** | - | 提供选品来源 | - |
| **内容库** | 关联推广的产品 | - | 提供待发布内容 |
| **发布中心** | 反馈转化数据 | 反馈内容表现 | - |

## 3. 产品库 (Products)

### 3.1 核心职责
**决策：哪些产品值得推广？**

### 3.2 产品状态（简化版）
```
待筛选 → 测试中 → 推广中 → 退出
pending  testing  active   phased_out
```

### 3.3 关键数据
- 产品基础信息（ASIN、标题、价格、评分）
- **关联内容数量** ← 新增：这个产品有多少篇内容在推广
- **总转化数据** ← 来自内容表现的汇总

### 3.4 与内容库的联动
- 产品详情页显示：该产品的所有关联内容
- 产品状态决策依据：关联内容的表现数据

## 4. 内容库 (Content)

### 4.1 核心职责
**创作：用什么内容来推广产品？**

### 4.2 内容类型
| 类型 | 说明 | 产品关联 |
|-----|------|---------|
| 产品评测 | 单品深度分析 | 1个产品 |
| 购买指南 | 多品对比推荐 | 多个产品 |
| 使用教程 | 场景化推荐 | 1-2个产品 |
| 清单文章 | Top 10榜单 | 多个产品 |
| 新闻资讯 | 行业动态+推荐 | 1-N个产品 |

### 4.3 内容生命周期
```
草稿 → 待审核 → 已通过 → 已发布 → 效果追踪
draft  pending  approved published tracking
```

### 4.4 内容卡片信息
```
┌────────────────────────────────────┐
│ 标题：Sony WH-1000XM4 深度评测   │
│ 类型：产品评测                    │
│ 关联产品：B08N5KWB9H (Sony耳机)  │
│ 状态：已发布                     │
├────────────────────────────────────┤
│ 📊 发布表现                      │
│ 发布平台：Blogger, Medium        │
│ 总浏览：1,234                    │
│ 点击数：89                       │
│ 转化数：5                        │
│ 收益：$125.50                    │
└────────────────────────────────────┘
```

### 4.5 与产品库的联动
- 创建内容时：选择要关联的产品（从产品库选择）
- 内容发布后：将表现数据汇总到关联产品

### 4.6 与发布中心的联动
- 内容审核通过后：进入发布队列
- 发布中心：执行多平台分发
- 发布结果：回写到内容记录

## 5. 发布中心 (Publishing)

### 5.1 核心职责
**分发：如何让内容触达更多受众？**

### 5.2 平台管理
| 平台 | 状态 | 配置 |
|-----|------|-----|
| Blogger | ✅ 已连接 | API Key已配置 |
| Medium | ✅ 已连接 | Integration Token |
| WordPress | ⚠️ 待配置 | 需要站点URL和密钥 |
| Twitter | ❌ 未连接 | - |

### 5.3 发布任务状态
```
待发布 → 发布中 → 成功/失败
pending  running success/failed
```

### 5.4 发布队列
```
┌─────────────────────────────────────────────────────────────┐
│ 发布队列                                                    │
├─────────────────────────────────────────────────────────────┤
│ 内容                          平台        状态      时间    │
│ Sony耳机评测                 Blogger     ✅成功    3/6 10:00│
│ Sony耳机评测                 Medium      ✅成功    3/6 10:05│
│ 2024耳机购买指南             Blogger     ⏳待发布  3/7 09:00│
│ Anker充电宝评测              Medium      ❌失败    3/6 11:00│
└─────────────────────────────────────────────────────────────┘
```

### 5.5 与内容库的联动
- 从内容库拉取"已通过"状态的内容
- 发布后更新内容状态为"已发布"
- 记录发布URL回写到内容

### 5.6 与产品库的联动
- 发布成功后，关联产品的"推广次数"+1
- 追踪数据回流到产品表现汇总

## 7. 核心功能清单

### 7.1 产品库功能
| 功能 | 优先级 | 说明 |
|-----|-------|------|
| AI推荐选品 | P0 | 基于市场数据的智能推荐 |
| 一键采集ASIN | P0 | 输入ASIN自动采集产品信息 |
| 产品状态管理 | P0 | pending/testing/active/phased_out |
| 关联内容展示 | P1 | 显示该产品的所有关联内容 |
| 表现数据汇总 | P1 | 来自内容的点击、转化、收益 |

### 7.2 内容库功能
| 功能 | 优先级 | 说明 |
|-----|-------|------|
| 素材采集 | P0 | 从Amazon/YouTube采集素材 |
| AI内容生成 | P0 | 基于素材自动生成内容 |
| 内容审核 | P0 | 人工审核/AI辅助审核 |
| **关联产品** | P0 | **核心：选择要推广的产品** |
| 发布状态追踪 | P1 | 显示内容在各平台的发布状态 |
| 表现数据展示 | P1 | 浏览、点击、转化、收益 |

### 7.3 发布中心功能
| 功能 | 优先级 | 说明 |
|-----|-------|------|
| 平台连接管理 | P0 | Blogger/Medium/WordPress配置 |
| 批量发布 | P0 | 一键发布到多平台 |
| 发布队列 | P0 | 查看发布任务状态 |
| 定时发布 | P1 | 预约发布时间 |
| 发布日志 | P1 | 成功/失败记录 |
| 表现数据拉取 | P2 | 从平台API获取数据 |

## 8. 关键改动点

### 8.1 内容库需要增加
1. **产品关联字段** - 每篇内容必须关联1+个产品
2. **内容类型字段** - 评测/指南/教程/清单
3. **发布状态** - 显示在哪些平台已发布
4. **表现数据** - 从发布平台回流的数据

### 8.2 产品库需要增加
1. **关联内容计数** - 有多少内容在推广这个产品
2. **表现数据汇总** - 来自所有关联内容的数据汇总
3. **简化状态** - 4个状态足够，不要太复杂

### 8.3 发布中心需要增加
1. **内容来源** - 从内容库选择，不是独立创建
2. **发布结果回写** - 成功后更新内容状态
3. **表现数据回流** - 定期从平台拉取数据

## 9. 实施计划

### Phase 1: 数据模型调整（0.5天）
- 内容表增加产品关联字段
- 内容表增加类型字段
- 产品表增加汇总统计字段

### Phase 2: 内容库改造（1天）
- 内容列表显示关联产品
- 内容创建/编辑时选择产品
- 内容详情显示发布状态和表现

### Phase 3: 发布中心改造（1天）
- 发布任务关联内容（而非独立）
- 发布成功后回写内容状态
- 表现数据定期同步

### Phase 4: 产品库联动（0.5天）
- 产品详情显示关联内容
- 产品表现数据汇总展示

### Phase 5: 联调测试（0.5天）
- 完整流程测试
- 数据一致性验证

## 10. 验收标准

### 产品库
- [ ] 可以看到每个产品关联了多少内容
- [ ] 可以看到产品的汇总表现数据（来自所有内容）
- [ ] AI推荐和采集功能正常

### 内容库
- [ ] 创建内容时可以选择关联产品
- [ ] 可以看到内容在哪些平台已发布
- [ ] 可以看到内容的点击/转化/收益数据

### 发布中心
- [ ] 发布时从内容库选择（而非手动输入）
- [ ] 发布成功后内容状态自动更新
- [ ] 可以看到每个发布任务的表现数据

### 整体联动
- [ ] 产品 → 内容 → 发布 的完整链路可追踪
- [ ] 数据在各模块间正确流转
